## Hallazgos iniciales

- La pagina principal afectada era `Views/Home/Menu.cshtml`.
- La vista usaba `Layout = null`, por lo que cargaba sus propios CSS y no pasaba por el layout compartido.
- El menu anterior dependia de muchos estilos inline:
  - anchos fijos de `450px`
  - alturas fijas de `80px`
  - `position-absolute` para la barra de usuario
  - botones Bootstrap grandes con poca adaptacion real a pantallas pequenas
- El CSS global ya tenia intentos responsive en `apple-design.css`, pero algunos dependian de selectores fragiles como `style*="width: 450px"`.
- El objetivo pedia una mejora de diseno completa, no solo pequenos ajustes visuales.

## Cambios implementados

### Rediseño de `Home/Menu.cshtml`

Se reemplazo la estructura anterior del menu por una composicion nueva:

- `main.menu-shell` como contenedor principal.
- `header.menu-topbar` para sesion actual, marca y salida.
- Marca visual `Juez de Altares` con icono.
- Bloque de usuario con nombre truncado de forma segura.
- Etiqueta de rol `Admin` o `Juez`.
- Boton `Cerrar sesion` con tamano tactil adecuado.
- Hero operativo `menu-workspace` con:
  - kicker `Control de evaluaciones`
  - titulo `Panel de control`
  - copy corto orientado al uso del concurso
  - imagen existente `catrina.png` como apoyo visual
- Acciones principales como tarjetas clicables:
  - `Nueva evaluacion`
  - `Consultar historial`
  - `Constancias`
- Zona administrativa con grid responsive:
  - `Jueces`
  - `Estudiantes historicos`
  - `Alumnos`
  - `Equipos`

La vista quedo sin estilos inline ni clases rigidas del menu anterior.

### CSS especifico para el menu

Se agrego un bloque nuevo en `AltarWeb/wwwroot/css/apple-design.css`:

- `body.tema-oscuro.menu-page`
- `.menu-shell`
- `.menu-topbar`
- `.menu-brand`
- `.menu-session`
- `.menu-workspace`
- `.menu-actions`
- `.menu-action`
- `.menu-admin`
- `.menu-admin-grid`
- `.menu-admin-link`

El diseno usa:

- superficies oscuras consistentes con el tema actual
- verde institucional para la accion principal
- azul y dorado como acentos secundarios para evitar una paleta de una sola nota
- bordes de 8px o menos en los elementos principales
- iconos FontAwesome ya presentes en el proyecto
- estados `hover` y `focus-visible`
- soporte de `prefers-reduced-motion`

### Responsive desktop/mobile

Se agregaron breakpoints especificos:

- `max-width: 900px`
  - acciones principales pasan de 3 columnas a 1 columna
  - admin pasa de 4 columnas a 2 columnas
- `max-width: 768px`
  - topbar se apila verticalmente
  - sesion usa grid compacto
  - hero pasa a una columna
  - tarjetas reducen altura y padding
- `max-width: 480px`
  - botones/tarjetas se vuelven mas compactos
  - admin pasa a una columna
  - flecha decorativa de tarjetas se oculta para ahorrar espacio
- `max-width: 360px`
  - ajustes especificos para pantallas ultra estrechas como iPhone SE
  - se oculta la imagen decorativa para evitar cortes y solapes
  - se reducen tamanos de texto y espacios
