### Gestion de equipos por alumnos

- Un alumno puede crear un equipo para el periodo actual.
- El creador queda como integrante y con `EsCreador = true`.
- Se valida que un alumno no pertenezca a mas de un equipo por periodo.
- Se agrego busqueda en tiempo real para seleccionar integrantes.
- Se agrego vista de equipo con integrantes, matricula, correo y rol.

## Incidencias durante la sesion

- El sandbox de escritura fallo varias veces con `windows sandbox: spawn setup refresh`; se usaron comandos escalados dentro del workspace.
- Habia un proceso `AltarWeb.exe` ejecutandose que bloqueaba `bin/Debug/net8.0`. Se detuvo para poder compilar y generar migraciones.
- Durante una correccion de migracion, EF removio/revirtio temporalmente una migracion historica. Se restauraron los archivos desde `HEAD` y se reaplico la migracion con `dotnet ef database update`.
- Se detectaron credenciales reales en `appsettings.json`; fueron reemplazadas por placeholders.

## Deuda tecnica

- `AlumnoEquipo` no tiene soft delete propio. La edicion admin puede remover membresias de la tabla puente. Si se requiere trazabilidad completa de cambios de integrantes, conviene agregar `Activo` a `AlumnoEquipo`.
- El login de jueces sigue usando el campo `Password` existente. No se migro para evitar romper cuentas actuales; conviene hacer un sprint dedicado para hashing de jueces.
- Las vistas nuevas son funcionales; una segunda pasada visual puede mejorar consistencia fina con el tema.
